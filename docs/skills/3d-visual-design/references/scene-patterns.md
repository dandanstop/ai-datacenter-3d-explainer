# Scene Patterns

## Shared Story Logic

Every 3D explainer scene should support three depths:

1. See the system
2. Touch the system
3. Read the system

The model should teach at depth 1. Labels and reading should support depth 2 and 3.

## Pattern 1: Exploded Infrastructure Stack

### Best Use

Use when the concept is a layered industrial system, physical stack, or supply chain.

### Proven Shape

A full environment visible on load, with selectable layers that move camera attention to one zone at a time.

### Good For

- Data center infrastructure
- Industrial systems
- Hardware supply chains
- Multi-layer process maps

### Keep

- Whole system visible first
- Clear numbered layers
- Subtle focus shifts after selection

## Pattern 2: Dual-Mode System Comparison

### Best Use

Use when one machine or platform behaves differently under two workloads or roles.

### Proven Shape

Left and right modes with a coordinating center.

### Good For

- Training vs inference
- Batch vs real-time systems
- Producer vs consumer flows

### Keep

- Clear orientation cues in overview
- Detailed internal labels only in selected mode
- Transparency increase in focus mode when internals matter

## Pattern 3: Character-Centered Workflow Scene

### Best Use

Use when a system concept becomes easier to understand through an actor, operator, or agent.

### Proven Shape

A central visual anchor plus a small number of supporting panels or devices.

### Good For

- Agentic AI
- Workflow automation
- Tool orchestration
- Human-system collaboration

### Keep

- One main anchor character or object
- Supporting workflow state panels
- Minimal connector lines
