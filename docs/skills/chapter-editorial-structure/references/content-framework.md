# Content Framework

## What a Strong Chapter Should Do

A strong chapter should help the reader:

1. Understand the system quickly
2. Understand the mechanism more deeply
3. Place companies and roles into that mechanism
4. Leave with a forward-looking question worth tracking

## Core Reading Layers

### 1. How it works

Use this to explain the mechanism plainly.

Questions to answer:

- What is this system doing?
- What kind of constraint shapes it?
- Which component or process makes it work?

### 2. Who builds this layer

Use this to place real companies into the stack.

Best practice:

- Group by function
- Use representative examples
- Add links when appropriate
- Keep the tone informational

### 3. Future signal

Use one sharp sentence or short paragraph.

This section should answer:

- What future shift could change the balance of this system?

### 4. Signals to watch

Use bullets when more than one signal matters.

Examples:

- Memory bandwidth becoming a bottleneck
- CPU coordination becoming more visible
- Power delivery slowing buildout
- Tool orchestration changing where value sits

### 5. Learn more

Use this for primary sources, company pages, or references that help readers go deeper.

## Audio Summary Rules

Audio should:

- Sound like a sharp narrated summary
- Compress the chapter into one memorable idea
- Keep sentence flow natural
- Avoid sounding like on-screen copy read aloud word for word

## Multilingual Rules

- Preserve teaching order
- Preserve technical accuracy
- Prefer natural phrasing over literal translation
- Keep headings consistent in function across languages
