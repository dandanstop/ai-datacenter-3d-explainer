# Chapter Editorial Structure

## Overview

Use this skill to structure chapter content for educational interactive explainers. It focuses on how a chapter teaches: the headline logic, section hierarchy, card structure, company references, future signals, audio transcript framing, and multilingual consistency.

Read [references/content-framework.md](references/content-framework.md) before rewriting chapter copy or designing a new reading layer.

## Use This Skill When

- A chapter has strong visuals but weak teaching value
- Insight cards feel generic, investment-led, or too technical
- The content needs a reusable chapter structure
- Company examples need to support education instead of stock framing
- Audio transcript, reading layer, and multilingual copy need to stay aligned

## Editorial Principles

1. Teach mechanism before opinion
   - Explain what the system does, how it works, and why it matters before adding interpretation.

2. Use company names as orientation, not hype
   - Readers should learn who participates in the system, not who to buy.

3. Future-facing content should be specific
   - Use `Future signal` and `Signals to watch` to point to concrete shifts, constraints, or enabling technologies.

4. Short copy for the model, richer copy for the reading layer
   - The 3D scene should not carry the essay.

5. Audio is a spoken summary, not a duplicate paragraph
   - Audio should sound natural, compressed, and memorable.

## House Structure

Use this structure when a chapter needs a stable reading system:

- `How it works`
- `Who builds this layer`
- `Future signal`
- `Signals to watch`
- `Learn more`

If a chapter is mode-based rather than layer-based, apply the same structure to each mode.

## Writing Sequence

1. Write the chapter thesis
   - One sentence that explains the chapter's teaching goal

2. Write the 5-second explanation
   - One short description for the first-view summary

3. Write the reading layer
   - Mechanism first
   - Role map second
   - Forward-looking interpretation third

4. Add company examples
   - Use real companies as reference points
   - Prefer linking them to function categories

5. Write audio summary
   - Spoken, concise, and aligned with the chapter thesis

6. Adapt for EN, ZH, KO, and JA
   - Preserve meaning hierarchy, not literal sentence shape

## Deliverables Checklist

- Chapter thesis
- First-view summary
- `How it works`
- `Who builds this layer`
- `Future signal`
- `Signals to watch`
- `Learn more`
- Audio summary
- Multilingual notes

## Common Mistakes

- Writing in an investment memo tone
- Listing company names before the system is explained
- Making every card sound structurally identical
- Treating audio as pasted body copy
- Translating literally instead of preserving clarity
